# Policies

This directory contains the canonical repository-operational rules for contributors and coding
agents.

Use these documents as the source of truth for how work should be performed in this repository.

## Policy Map

- [engineering.md](engineering.md) — code authoring, typing, hygiene, and pre-release fallback
  policy
- [documentation.md](documentation.md) — docs placement, claim discipline, and documentation truth
- [architecture.md](architecture.md) — architectural biases, ADR expectations, and canonical design
  reading
- [verification.md](verification.md) — testing, evidence, and verification expectations

## Relationship To Other Documentation

- Product and end-user docs live in [`../docs/`](../docs/).
- Design authority and design history live in [`../design/`](../design/).
- `AGENTS.md` is the short entrypoint that routes here.
